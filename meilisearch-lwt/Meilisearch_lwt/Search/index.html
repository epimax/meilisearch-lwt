Module Meilisearch_lwt.Search

Module to perform searches on a Meilisearch instance. Provides helpers for the different types of searches.

Search module of the library. We provide two submodules, Query and Result, to help building queries and reading results. There is no support for media search or enterprise-only functionality.

type search_error = [
  1. | Meilisearch_lwt__.Meilisearch_error.t
  2. | `NoSuchIndex of string
]

Errors that can occur in this module: index not found, and also network/system errors.

See examples/ for suggested workflows. Since the response from Meilisearch changes depending on your query, we provide this module that reads the JSON manually to give you back the hits field and other mandatory fields from the response. Since the library doesn't enforce any schema validation on the documents you insert, it doesn't help you read the results either. You can define your own derivable record that reads the hits field elements, or read them manually.

val search_term : Meilisearch_lwt__.Meilisearch_client.t -> index_uid:string -> q:string -> (Yojson.Safe.t, search_error) Stdlib.result Lwt.t

search_term client ~index_uid ~q sends a simple term search to your instance. This is a GET request to Meilisearch that looks in every searchableAttributes of your document.

val search_post : Meilisearch_lwt__.Meilisearch_client.t -> index_uid:string -> Query.t -> (Yojson.Safe.t, search_error) Stdlib.result Lwt.t

search_post client ~index_uid query sends a structured search to your instance. This is a POST request to Meilisearch. The query should be built with the submodule Query.

val search_post_raw : Meilisearch_lwt__.Meilisearch_client.t -> index_uid:string -> Yojson.Safe.t -> (Yojson.Safe.t, search_error) Stdlib.result Lwt.t

search_post_raw client ~index_uid query sends a raw structured search to your instance. This is a POST request to Meilisearch. The query is a JSON object built by yourself that represents a valid request to Meilisearch, with no checks performed on the library side.

val search_term_idx : Meilisearch_lwt__.Meilisearch_client.t -> Meilisearch_lwt__.Meilisearch_index.index -> q:string -> (Yojson.Safe.t, search_error) Stdlib.result Lwt.t

search_term_idx client index ~q sends a simple term search to your instance. The _idx APIs use the internal Meilisearch_lwt.Index.index record to ensure the index exists on your instance. This is a GET request to Meilisearch that looks in every searchableAttributes of your document.
