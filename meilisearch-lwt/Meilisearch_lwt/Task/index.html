Module Meilisearch_lwt.Task

Module for monitoring tasks. Many of the modules described below return a task ID; the Task submodule provides helpers to wait for or check these tasks.

Task management. Meilisearch works in batches, and since these are queued you might not always get a success/failure response from the server immediately: the task API will let you know.

type task_error = [
  1. | Meilisearch_lwt__.Meilisearch_error.t

Errors that can occur in this module: task failures, timeouts while waiting, and also network/system errors.

val wait_task : Meilisearch_lwt__.Meilisearch_client.t -> ?max_attempts:int -> delay:float -> int -> (task, task_error) Stdlib.result Lwt.t

wait_task client ?max_attempts ~delay task_id returns a promise that resolves when the task is completed, canceled, or has timed out according to the max_attempts provided or the instance behavior. Delay is expressed in seconds.

val check_task : Meilisearch_lwt__.Meilisearch_client.t -> task_uid:int -> (task_status, task_error) Stdlib.result Lwt.t

check_task client ~task_uid returns the instantaneous status of the task. If you want the full task details, you need to call get_task.
